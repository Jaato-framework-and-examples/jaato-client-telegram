# File Sharing Implementation

## Overview

This implementation adds file sharing capabilities to `jaato-client-telegram`, allowing the bot to send files generated by the jaato AI agent to Telegram users via `file.generated` events from the SDK.

The delivery method is **size-based**:
- **Small files** (< `link_threshold_kb`, default 100KB) are sent as **document attachments**
- **Large files** (>= `link_threshold_kb`) are sent using **Telegram's file hosting URL** (download links)

## Components Added

### 1. `FileSharingConfig` Class (`config.py`)

```python
class FileSharingConfig(BaseModel):
    """File sharing configuration for generated files."""

    enabled: bool = True
    max_file_size_mb: int = 10  # Telegram bot limit
    allowed_extensions: list[str] = Field(
        default_factory=lambda: [".txt", ".csv", ".json", ".xml", ".yaml", ".yml", ".md", ".py", ".js", ".ts", ".html", ".css"]
    )
    # Size threshold for determining delivery method (in KB)
    # Files < this size are sent as document attachments
    # Files >= this size are sent as Telegram file hosting URLs
    link_threshold_kb: int = 100  # 100KB default threshold
```

**Configuration Options:**
- `enabled`: Toggle file sharing on/off (default: `true`)
- `max_file_size_mb`: Maximum file size in MB (default: 10, Telegram's limit is 50MB)
- `allowed_extensions`: List of permitted file extensions (lowercase with dots)
- `link_threshold_kb`: Size threshold in KB that determines delivery method (default: 100)

### 2. `FileHandler` Class (`file_handler.py`)

A new module that handles `file.generated` events from the SDK:

**Key Methods:**
- `handle_file_event(event, message)`: Processes file generation events and sends files to Telegram
- `_send_as_document(file_path, message)`: Sends file as a Telegram document attachment
- `_send_as_link(file_path, message)`: Sends file using Telegram's file hosting URL

**Size-Based Delivery Logic:**
```python
# Determine how to send the file based on size
if file_size_kb < self._config.link_threshold_kb:
    # Small file - send as document attachment
    await self._send_as_document(file_path_obj, message)
else:
    # Large file - send using Telegram's file hosting URL
    await self._send_as_link(file_path_obj, message)
```

**Features:**
- File validation (existence, size, extension)
- **Size-based delivery strategy** (attachments for small, links for large)
- Smart link extraction from uploaded documents
- Graceful error handling with user-friendly messages
- Security: Uses filename only (not full path) for uploads
- Automatic truncation for long text files to fit Telegram's 4096 char limit

### 3. Integration with `ResponseRenderer` (`renderer.py`)

Added support for `file.generated` events:

```python
elif event_type == "file.generated" or event_type == "FILE_GENERATED":

    if self._file_handler:

**Behavior:**
- Flushes any pending text output before sending the file
- Sends file based on size (attachment or link)
- Continues streaming events after file is sent
- Logs success/failure of file operations

## Event Flow

### For Small Files (< link_threshold_kb)

1. **jaato SDK** emits a `file.generated` event
2. **ResponseRenderer** receives the event and flushes any pending text
3. **FileHandler** validates the file (size, extension, existence)
4. **FileHandler** calls `_send_as_document()` to upload as attachment
5. **Telegram** receives and displays the document with caption

### For Large Files (>= link_threshold_kb)

1. **jaato SDK** emits a `file.generated` event
2. **ResponseRenderer** receives the event and flushes any pending text
3. **FileHandler** validates the file (size, extension, existence)
4. **FileHandler** calls `_send_as_link()` which:
   - Uploads the file to Telegram via `answer_document()`
   - Extracts the file URL from the uploaded document using `document.get_url()`
   - Sends a follow-up message with the download link
5. **Telegram** stores the file and provides a download URL
6. **User** receives two messages:
   - First: Document attachment with "Uploading file..." caption
   - Second: Message with filename and download link

**Example User Experience for Large File:**
```
Bot: 📄 Uploading file: data.csv...
Bot: 📄 Generated file: `data.csv`

🔗 Download link: https://t.me/botname?file=abc123...
```

## Error Handling

The implementation handles various error scenarios:

- **File not found**: User receives warning message with file path
- **File too large**: User receives warning with actual size and limit
- **Invalid extension**: User receives warning with disallowed extension
- **Send failure**: User receives error message with exception details
- **Link extraction failure**: User receives fallback message if URL cannot be retrieved
- **Permission errors**: Logged but don't crash the bot

## Security Considerations

1. **Path sanitization**: Only the filename (not full path) is used for document uploads
2. **Extension validation**: Only whitelisted extensions are allowed
3. **Size limits**: Enforces configurable size limits to prevent abuse
4. **Error messages**: Don't expose sensitive system paths in user-facing errors

## Size-Based Delivery Strategy

### Why Two Methods?

**Small Files as Attachments (< 100KB):**
- ✅ Instant access - file appears directly in chat
- ✅ Works offline - no internet needed to view
- ✅ Familiar UX - standard Telegram document behavior
- ✅ No storage dependency on Telegram's servers beyond attachment

**Large Files as Links (>= 100KB):**
- ✅ Faster delivery - no need to wait for download in chat
- ✅ Better for bandwidth - user can choose when to download
- ✅ Works around attachment size limits (some platforms have limits)
- ✅ Shareable - link can be shared with others
- ✅ Persists on Telegram's servers - doesn't expire immediately

### Configurable Threshold

The `link_threshold_kb` setting lets you tune the behavior:

- **Lower threshold (e.g., 50KB)**: More files sent as links, better for slow connections
- **Higher threshold (e.g., 500KB)**: More files sent as attachments, better for instant access

Default is 100KB as a good balance between instant access and delivery efficiency.

The user receives:

- The file as a document attachment (instantly accessible)

       [Uploads to Telegram]

🔗 Download link: https://t.me/botname?file=xyz789...
```

The user receives:
1. The agent's text response
2. Document upload message with "Uploading..." caption
3. Download link message with filename and URL

## Testing

To test file sharing:

1. Ensure `file_sharing.enabled: true` in your config
2. Set `file_sharing.link_threshold_kb` to desired threshold (default: 100)
3. Test with a small file (< threshold): Should receive as attachment
4. Test with a large file (>= threshold): Should receive upload + download link
5. Verify file size limits are enforced
6. Test with different file extensions (allowed vs disallowed)
